# Concepts

The toolkit uses four core concepts.

## Legibility

Legibility is the degree to which a capable engineer or agent can understand why a capability behaves the way it does without relying on tribal knowledge.

## Capability

A capability is a business-facing unit of behavior with inputs, outputs, non-goals, and a human-readable name that can be referenced in contracts and graphs.

## Contract

A contract is the machine-readable and human-readable declaration of a capability. It says what the capability does, what it needs, what it promises, and what it explicitly does not do.

## Capability Graph

The capability graph is the directed dependency map of all contracts in the repo. It shows how capabilities connect, where change can propagate, and which areas still need extraction work.
