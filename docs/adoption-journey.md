# Adoption Journey

The Day After Toolkit turns one diagnostic into a workflow. This guide follows the path from first inspection to steady-state maintenance.

## The Six Stages

1. `cdad check` establishes the baseline by combining a static scan with the legibility questionnaire.
2. `cdad roadmap` prioritizes the capabilities that most need extraction.
3. `cdad init` scaffolds a contract for one capability at a time.
4. `cdad validate` keeps the contract files, generated artifacts, and policy rules honest.
5. `cdad graph` makes the capability topology visible to humans and agents.
6. `cdad check` runs again to measure whether the repo is becoming easier to navigate.

## Day One

Start with `cdad check` in the repository root, then move straight to `cdad roadmap` for the highest-risk capabilities. The point is not to document everything at once; it is to make the next safe change obvious.
