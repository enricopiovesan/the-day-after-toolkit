# Adoption Journey

This is the six-stage path from checking a repository to maintaining its contracts over time.

1. Discover with `cdad check`.
2. Plan with `cdad roadmap`.
3. Execute with `cdad init`.
4. Validate with `cdad validate`.
5. Visualize with `cdad graph`.
6. Maintain by running `cdad check` again after each extraction phase.

The point of the journey is not to produce more files. It is to make the codebase easier for a first-time practitioner or agent to navigate safely.
