# Security Policy

## Supported Stage

The Day After Toolkit should currently be treated as pre-production software.

## Reporting a Vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Instead, report them privately to the maintainer using the support path in [SUPPORT.md](/Users/piovese/Documents/the-day-after-toolkit/SUPPORT.md) and clearly mark the report as a security issue.

## What to Include

Please include:

- affected area
- reproduction steps
- impact
- any known mitigation

## Security Direction

The project is designed toward:

- validated and versioned artifacts
- explicit contract boundaries
- machine-readable outputs for automated checks
- provenance-aware documentation and workflows
- safe defaults when agent context is incomplete
